# 🧱 REQUIRED FILE TEMPLATE (MANDATORY)

Every markdown file must follow this structure:

## 1. Purpose
- What problem does this file solve?
- Why does it exist?
- What breaks if removed?

## 2. Architectural Layer
Specify:
- Settings / Foundation / Layout / Runtime / Elements / Scenarios / Utilities
Explain why it belongs there.

## 3. Core Concept
Explain:
- What CSS mechanism it uses (grid, flex, flow, positioning, etc.)
- Why chosen
- Alternatives considered

## 4. CSS Fundamentals (MANDATORY)
Teach CSS for every feature used in this file (e.g. @layer, :where, custom properties, var(), display:flex, grid-template-columns, gap, logical properties, etc.)
Explain:
- What it is
- How it works internally
- When it is evaluated
- Cascade impact
- Specificity impact
- Inheritance behavior
- Computed vs specified value (when relevant)
- Edge cases
- Browser behavior assumptions

Assume the reader does NOT fully understand CSS.

## 5. CSS Properties Breakdown
For each CSS property used in this file, explain:
- What it does
- Why used here
- What problem it solves
- Interactions with other properties
- Common mistakes
- Edge cases

## 6. MILOS Implementation Logic
Explain:
- How MILOS uses these mechanisms
- What restrictions MILOS imposes
- What is intentionally not allowed
- How this fits into the engine architecture

## 7. Interaction Matrix
Provide a table:
| Works With | Conflicts With | Not Intended With |
Explain interactions with other primitives.

## 8. Nesting & Collapse Behavior
Explain behavior when:
- nested inside itself
- nested inside grid
- nested inside stack
- combined with region
- combined with bleed
- combined with runtime classes

## 9. Diagram (MANDATORY)
Provide ASCII diagram illustrating:
- box model / tracks / axes / positioning / overflow (as relevant)

## 10. Valid Example
Provide minimal valid HTML usage.

## 11. Invalid Example
Show misuse and explain why incorrect.

## 12. Boundaries
Explicitly state:
- what this file does NOT do
- what problem it does NOT solve
- which layer handles that instead

## 13. Engine Decision Log
Explain:
- why this solution was chosen
- alternatives rejected
- why it is core / element / scenario
- whether it might be considered sugar
