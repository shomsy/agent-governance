# `projects/` — Temporary Knowledge Base for Generalization

This folder contains a collection of governance, architecture, and documentation files gathered from various projects (`AVAX Bootcamp`, `Baraba`, `Polymoly`, `Admin Dashboard`, etc.).

## 🎯 Purpose

The contents of this folder are **not** the active governance rules for this repository. Instead, they serve as the **raw material, patterns, and inspiration** for building and refining the universal [`.agents/`](../.agents/) folder.

Every rule, "Naming Law", and architectural standard currently residing in `.agents/` started its life as a successful pattern in one of these project-specific folders.

## 🛠️ Usage for AI & Human Contributors

1.  **Analyze**: Look at the `AGENTS.md` and `docs/governance/` subdirectories in these projects to find effective ways of describing work.
2.  **Generalize**: When you find a rule that works in more than one project (e.g., the "Flow -> Responsibility -> Action" rule), extract it, generalize the language, and move it into the core [`.agents/`](../.agents/) folder.
3.  **Refine**: Use these real-world examples to test if a new global rule is too abstract or too rigid.

## 🕰️ Final Destination

Once a project-specific pattern is successfully integrated into the universal OS hub, the reference in this folder becomes a historical artefact. This directory will eventually be retired or cleaned of files that have been fully generalized.
