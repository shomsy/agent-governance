# HotelSync ↔ BridgeOne: Integration Plan

This document is the project-level map for the integration service.

Its job is simple:

- explain what the service must do
- show how the current repository solves it
- make the implementation easy to review without reverse-engineering the whole codebase

The original interview brief is kept in [`INTERVIEW_TASK.md`](INTERVIEW_TASK.md).

---

## Mission Snapshot

The service connects:

- HotelSync as the upstream source of truth
- BridgeOne as the downstream local integration target

The repository implements a procedural PHP 8 service with:

- feature-first business folders
- `mysqli` for database access
- `cURL` for outbound HTTP
- MySQL as the persistence layer
- Docker for reproducible local verification

### Quick read

- `app/` contains technical runtime code
- `features/` contains business flows
- `public/webhooks/otasync.php` is the inbound webhook entrypoint
- `setup/database/create_schema.sql` defines the local schema
- `tests/` proves mapping rules, idempotency, retry behavior, and webhook deduplication

## Step 1: Catalog Sync

The catalog flow is responsible for stable local room and rate-plan mappings.

Main files:

- `sync_catalog.php`
- `features/catalog/commands/run_catalog_sync_command.php`
- `features/catalog/sync_catalog_from_hotelsync.php`
- `features/catalog/rooms/fetch_rooms_from_hotelsync.php`
- `features/catalog/rooms/transform_room_payload_to_catalog_record.php`
- `features/catalog/rooms/upsert_room_catalog_record.php`
- `features/catalog/rate_plans/fetch_boards_from_hotelsync.php`
- `features/catalog/rate_plans/fetch_rate_plans_from_hotelsync.php`
- `features/catalog/rate_plans/transform_boards_payload_to_board_names_by_id.php`
- `features/catalog/rate_plans/transform_rate_plan_payload_to_catalog_record.php`
- `features/catalog/rate_plans/upsert_rate_plan_catalog_record.php`

What it does:

- logs in to HotelSync
- fetches boards because the API exposes meal-plan semantics there
- fetches rooms
- fetches pricing plans
- generates stable local codes
- compares canonical payload hashes
- upserts only changed rows

```mermaid
flowchart TD
    classDef step1 fill:#E8F5E9,stroke:#2E7D32,stroke-width:2px,color:#1B5E20;

    A["Step 1.1<br/>Login to HotelSync"]:::step1 --> B["Step 1.2<br/>Fetch boards"]:::step1
    B --> C["Step 1.3<br/>Fetch rooms"]:::step1
    B --> D["Step 1.4<br/>Fetch rate plans"]:::step1
    C --> E["Step 1.5<br/>Build clean local records"]:::step1
    D --> E
    E --> F["Step 1.6<br/>Compare payload hashes"]:::step1
    F --> G["Step 1.7<br/>UPSERT catalog rows"]:::step1

    linkStyle 0,1,2,3,4,5,6 stroke:#2E7D32,stroke-width:2px;
```

### Quick read

- room code format is `HS-{ROOM_ID}-{slug_room_name}`
- rate-plan code format is `RP-{RATE_PLAN_ID}-{meal_plan}`
- canonical hashes avoid pointless updates
- unique DB constraints keep mappings stable

### Step by step

- 🟩 Step 1.1: We log in to HotelSync so the catalog flow can start.
- 🟩 Step 1.2: We fetch boards because meal plan names live there.
- 🟩 Step 1.3: We fetch room payloads from the API.
- 🟩 Step 1.4: We fetch rate-plan payloads from the API.
- 🟩 Step 1.5: We build clean local records from the raw payloads.
- 🟩 Step 1.6: We compare payload hashes against local state.
- 🟩 Step 1.7: We UPSERT only the catalog rows that changed.

### How this works

```mermaid
sequenceDiagram
    participant CMD as Catalog Command
    participant HS as HotelSync API
    participant DB as Local Database

    CMD->>HS: 1.1 POST /login
    HS-->>CMD: Access token and property context
    CMD->>HS: 1.2 GET /boards
    HS-->>CMD: Board payload
    CMD->>HS: 1.3 GET /rooms
    HS-->>CMD: Room payloads
    CMD->>HS: 1.4 GET /rate-plans
    HS-->>CMD: Rate-plan payloads
    CMD->>CMD: 1.5 Build local room and rate-plan records
    CMD->>DB: 1.6 Read current hashes and local ids
    CMD->>DB: 1.7 UPSERT only changed catalog rows
```

The catalog command starts by opening a valid HotelSync session. After that it reads boards, rooms, and rate plans, converts those upstream payloads into the local catalog shape, and checks the stored hashes in MySQL before writing anything. This keeps the flow idempotent and avoids rewriting rows that did not change.

## Step 2: Reservation Import

The reservation import flow syncs date-ranged upstream reservations into the local model.

Main files:

- `sync_reservations.php`
- `features/reservations/commands/run_reservation_import_command.php`
- `features/reservations/import/fetch_reservations_from_hotelsync_by_date_range.php`
- `features/reservations/import/sync_reservations_from_hotelsync_by_date_range.php`
- `features/reservations/local_state/transform_reservation_payload_to_local_record_set.php`
- `features/reservations/local_state/save_reservation_record_set_to_local_database.php`
- `features/reservations/local_state/sync_reservation_payload_to_local_state.php`

What it does:

- fetches reservations by `--from` and `--to`
- maps one reservation into a parent row plus child room and rate-plan rows
- generates `LOCK-{reservation_id}-{arrival_date}`
- compares hashes to skip unchanged payloads
- replaces child snapshots when the payload changes
- writes audit entries for imported, updated, unchanged, and canceled states

```mermaid
flowchart TD
    classDef step2 fill:#E3F2FD,stroke:#1565C0,stroke-width:2px,color:#0D47A1;

    A["Step 2.1<br/>Read --from and --to"]:::step2 --> B["Step 2.2<br/>Fetch reservations"]:::step2
    B --> C["Step 2.3<br/>Build local record sets"]:::step2
    C --> D["Step 2.4<br/>Generate lock_id values"]:::step2
    D --> E["Step 2.5<br/>Compare payload hashes"]:::step2
    E --> F["Step 2.6<br/>Save reservation records"]:::step2
    F --> G["Step 2.7<br/>Write audit rows"]:::step2

    linkStyle 0,1,2,3,4,5 stroke:#1565C0,stroke-width:2px;
```

### Quick read

- one reservation may contain many rooms
- one reservation may contain many rate plans
- local child rows are treated as a fresh snapshot of the latest payload
- unchanged hashes become a no-op instead of a duplicate write

### Step by step

- 🟦 Step 2.1: We read the date range from the CLI command.
- 🟦 Step 2.2: We fetch reservation payloads for that period.
- 🟦 Step 2.3: We split each reservation into local parent and child records.
- 🟦 Step 2.4: We generate `lock_id` values for local tracking.
- 🟦 Step 2.5: We compare payload hashes to detect real changes.
- 🟦 Step 2.6: We save the reservation records into the local schema.
- 🟦 Step 2.7: We write audit rows for the result of that sync.

### How this works

```mermaid
sequenceDiagram
    participant CMD as Reservation Sync Command
    participant HS as HotelSync API
    participant DB as Local Database

    Note over CMD: 2.1 Read --from and --to
    CMD->>HS: 2.2 GET /reservations for the date range
    HS-->>CMD: Reservation payloads
    loop For each reservation
        CMD->>CMD: 2.3 Build parent and child record set
        CMD->>CMD: 2.4 Generate LOCK-{reservation_id}-{arrival_date}
        CMD->>DB: 2.5 Read current reservation hash
        CMD->>DB: 2.6 Save reservation, rooms, and rate plans
        CMD->>DB: 2.7 Insert audit row
    end
```

The reservation import command reads a date range and asks HotelSync for all reservations inside that window. For each payload it builds one local reservation snapshot, generates the local `lock_id`, compares the current hash, and only then writes the parent row, child rows, and audit trace. The database becomes the durable record of what was imported and what changed.

## Step 3: Reservation Update / Cancel

The single-reservation update flow reuses the same local-state logic as the import flow.

Main files:

- `update_reservation.php`
- `features/reservations/commands/run_reservation_update_command.php`
- `features/reservations/update/fetch_reservation_from_hotelsync_by_id.php`
- `features/reservations/update/sync_reservation_from_hotelsync_by_id.php`
- `features/reservations/local_state/detect_cancelled_reservation_from_payload.php`
- `features/reservations/local_state/load_or_sync_reservation_by_external_id.php`

What it does:

- fetches one reservation by external id
- checks whether the reservation already exists locally
- compares payload hashes
- updates only when there is an actual change
- keeps canceled reservations in place instead of deleting them
- writes audit records for state transitions

```mermaid
flowchart TD
    classDef step3 fill:#FFF3E0,stroke:#EF6C00,stroke-width:2px,color:#E65100;

    A["Step 3.1<br/>Read reservation id"]:::step3 --> B["Step 3.2<br/>Fetch one reservation"]:::step3
    B --> C["Step 3.3<br/>Load local reservation"]:::step3
    C --> D["Step 3.4<br/>Compare hash and status"]:::step3
    D --> E["Step 3.5<br/>Apply update or cancel state"]:::step3
    E --> F["Step 3.6<br/>Write audit event"]:::step3

    linkStyle 0,1,2,3,4 stroke:#EF6C00,stroke-width:2px;
```

### Quick read

- cancel is treated as a business event, not a hard delete
- local state remains auditable
- the same sync logic is reused by CLI update flow and webhook flow

### Step by step

- 🟧 Step 3.1: We read the reservation id from the CLI command.
- 🟧 Step 3.2: We fetch that one reservation from HotelSync.
- 🟧 Step 3.3: We load the local reservation if it already exists.
- 🟧 Step 3.4: We compare the new hash and status with local state.
- 🟧 Step 3.5: We apply an update or cancel state only when needed.
- 🟧 Step 3.6: We write an audit event for the final state change.

### How this works

```mermaid
sequenceDiagram
    participant CMD as Update Command
    participant HS as HotelSync API
    participant DB as Local Database

    Note over CMD: 3.1 Read reservation_id
    CMD->>HS: 3.2 GET /reservation/{id}
    HS-->>CMD: Reservation payload
    CMD->>DB: 3.3 Load local reservation state
    CMD->>CMD: 3.4 Compare payload hash and cancel status
    alt Payload changed or reservation canceled
        CMD->>DB: 3.5 Update local reservation state
        CMD->>DB: 3.6 Insert audit event
    else Unchanged payload
        CMD-->>CMD: Skip write and return unchanged
    end
```

This flow is narrow on purpose: it works on one reservation only. The command fetches the latest upstream payload, checks the local version, and decides whether the record is unchanged, updated, or canceled. A cancellation is stored as a state change, not as a delete, so the reservation history stays visible.

## Step 4: Invoice Flow

The invoice flow turns one synced reservation into one queued invoice payload.

Main files:

- `generate_invoice.php`
- `features/invoices/commands/run_invoice_generation_command.php`
- `features/invoices/payload/transform_reservation_payload_to_invoice_line_items.php`
- `features/invoices/payload/transform_local_reservation_to_invoice_payload.php`
- `features/invoices/queue/find_invoice_queue_item_by_reservation_id.php`
- `features/invoices/queue/find_invoice_queue_item_by_id.php`
- `features/invoices/queue/reserve_next_invoice_number.php`
- `features/invoices/queue/save_invoice_queue_item.php`
- `features/invoices/queue/mark_invoice_queue_item_as_processing.php`
- `features/invoices/queue/mark_invoice_queue_item_as_sent.php`
- `features/invoices/queue/mark_invoice_queue_item_for_retry.php`
- `features/invoices/generate_and_queue_invoice_for_reservation.php`
- `features/invoices/delivery/deliver_invoice_from_queue.php`

What it does:

- loads or synchronizes the reservation first
- reserves the next invoice number atomically
- builds line items from the stored reservation payload
- prepares the BridgeOne payload
- inserts or reuses the queue row
- sends the invoice to the configured endpoint
- retries failed delivery up to the configured limit

```mermaid
flowchart LR
    classDef step4 fill:#F3E5F5,stroke:#7B1FA2,stroke-width:2px,color:#4A148C;

    A["Step 4.1<br/>Load or sync reservation"]:::step4 --> B["Step 4.2<br/>Lock invoice counter"]:::step4
    B --> C["Step 4.3<br/>Generate invoice number"]:::step4
    C --> D["Step 4.4<br/>Build invoice payload"]:::step4
    D --> E["Step 4.5<br/>Insert or reuse queue row"]:::step4
    E --> F["Step 4.6<br/>Deliver invoice payload"]:::step4
    F --> G["Step 4.7<br/>Mark retry or failure state"]:::step4

    linkStyle 0,1,2,3,4,5 stroke:#7B1FA2,stroke-width:2px;
```

### Quick read

- numbering is DB-backed, not in-memory
- queue rows are unique per reservation
- failures move through retry state and stop at `failed`
- HTTPS is required for outbound invoice delivery by default

### Step by step

- 🟪 Step 4.1: We load or synchronize the reservation before invoicing it.
- 🟪 Step 4.2: We lock the invoice counter in the database.
- 🟪 Step 4.3: We generate the next unique invoice number.
- 🟪 Step 4.4: We build the full invoice payload.
- 🟪 Step 4.5: We insert or reuse the queue row for delivery.
- 🟪 Step 4.6: We send the invoice payload to the configured endpoint.
- 🟪 Step 4.7: We mark retry or failure state when delivery does not succeed.

### How this works

```mermaid
sequenceDiagram
    participant CMD as Invoice Command
    participant DB as Local Database
    participant BO as BridgeOne Endpoint

    CMD->>DB: 4.1 Load or sync reservation
    CMD->>DB: 4.2 SELECT invoice_counters FOR UPDATE
    DB-->>CMD: Current counter state
    CMD->>DB: 4.3 Update counter and reserve invoice number
    CMD->>CMD: 4.4 Build invoice payload
    CMD->>DB: 4.5 Insert or reuse invoice_queue row
    CMD->>BO: 4.6 POST invoice payload
    alt Delivery failed
        CMD->>DB: 4.7 Increase retry count or mark failed
    else Delivery succeeded
        CMD->>DB: Mark queue row as sent
    end
```

The invoice command uses the local reservation as its source data. It reserves the invoice number inside MySQL with a locked counter, prepares the outbound payload, stores the queue row, and then attempts delivery to the configured BridgeOne endpoint. If delivery fails, the queue state is updated so retries remain controlled and visible.

## Step 5: Webhook Flow

The webhook flow is the inbound integration surface.

Main files:

- `public/webhooks/otasync.php`
- `features/webhooks/reservations/receive_reservation_webhook_request.php`
- `features/webhooks/reservations/transform_webhook_payload_to_webhook_context.php`
- `features/webhooks/reservations/manage_webhook_event_inbox.php`
- `features/webhooks/reservations/process_reservation_webhook_event.php`

What it does:

- accepts only `POST`
- accepts only JSON content types
- rejects oversized payloads
- parses the webhook body
- derives a dedupe key from upstream event id or canonical payload hash
- stores the inbox event in `webhook_events`
- reuses reservation sync logic
- returns duplicate-safe responses when the same event arrives again

```mermaid
flowchart TD
    classDef step5 fill:#FCE4EC,stroke:#C2185B,stroke-width:2px,color:#880E4F;

    A["Step 5.1<br/>Receive POST /webhooks/otasync.php"]:::step5 --> B["Step 5.2<br/>Validate method, JSON, and size"]:::step5
    B --> C["Step 5.3<br/>Parse JSON body"]:::step5
    C --> D["Step 5.4<br/>Build context and dedupe key"]:::step5
    D --> E["Step 5.5<br/>Store webhook event"]:::step5
    E --> F["Step 5.6<br/>Reuse reservation sync logic"]:::step5
    F --> G["Step 5.7<br/>Mark final webhook status"]:::step5

    linkStyle 0,1,2,3,4,5 stroke:#C2185B,stroke-width:2px;
```

### Quick read

- if upstream sends an event id, that becomes the primary dedupe key
- if upstream does not send one, the payload hash becomes the fallback dedupe key
- duplicate webhook delivery stays harmless
- transport validation is implemented; signed sender verification is intentionally documented as a next production step

### Step by step

- 🟥 Step 5.1: HotelSync sends a message to the webhook URL.
- 🟥 Step 5.2: We fail closed unless the request method, JSON type, and size are valid.
- 🟥 Step 5.3: We parse the request body into readable data.
- 🟥 Step 5.4: We build the context and the dedupe key.
- 🟥 Step 5.5: We store the incoming event in the inbox table.
- 🟥 Step 5.6: We reuse reservation sync logic instead of duplicating reservation update code.
- 🟥 Step 5.7: We mark the event as processed, duplicate, or failed.

### How this works

```mermaid
sequenceDiagram
    participant HS as HotelSync
    participant WH as Webhook Endpoint
    participant DB as Local Database

    HS->>WH: 5.1 POST webhook event
    WH->>WH: 5.2 Validate method, JSON type, and size
    WH->>WH: 5.3 Parse JSON body
    WH->>WH: 5.4 Build context and dedupe key
    WH->>DB: 5.5 Read or insert webhook_events row
    alt Duplicate event
        DB-->>WH: Existing dedupe key found
        WH-->>HS: 5.7 200 OK duplicate
    else New event
        WH->>WH: 5.6 Reuse reservation sync logic
        WH->>DB: Update reservation state and webhook status
        WH-->>HS: 5.7 200 OK processed
    end
```

The webhook endpoint behaves like an inbound inbox. It first validates the transport layer, then turns the request into a dedupe-aware event record, and only after that does it touch reservation state. If the same event arrives again, the database dedupe key lets the service return a safe response without creating duplicate updates.

## Security Posture

This repository is interview-scoped, but it is not naive.

Implemented safeguards:

- prepared statements through `mysqli`
- canonical payload hashing for idempotency decisions
- webhook method validation
- webhook content-type validation
- webhook payload size limit
- duplicate suppression through DB-backed dedupe keys
- HTTPS enforcement for invoice delivery unless explicitly relaxed
- correlation ids propagated through CLI entrypoints, webhook requests, logs, and outbound HTTP headers
- `nginx` exposes only the webhook entrypoint from `public/`
- code is mounted read-only into the PHP containers during Docker verification
- secrets are not written into logs

OWASP-minded interpretation:

- fail closed on malformed or unexpected webhook requests
- minimize exposed HTTP surface
- avoid duplicate state transitions under replay
- keep a durable audit trail for operational review
- keep outbound delivery explicit and configurable

### Quick read

- this is not full production hardening
- it is a deliberate integration-safe baseline
- signature verification and rate limiting are documented as next steps, not silently ignored

### Small performance notes

- no Redis or external cache layer was added
- reservation persistence now loads only the room ids and rate plan ids referenced by the current payload
- helper indexes were added only where the repository already has obvious read patterns
- the database remains the source of truth for correctness, not a cache layer

## Design Decisions and Trade-offs

- The project stays procedural because the task explicitly asks for procedural PHP and forbids framework-style HTTP/DB abstraction.
- The code uses feature-first folders because the evaluator should be able to find the business flow quickly.
- `boards` are fetched during catalog sync because meal-plan naming comes from that upstream concept.
- Webhook dedupe prefers upstream event ids and falls back to canonical payload hashes when needed.
- Webhook inbox persistence reuses the same row on the unique dedupe key, so replay races stay duplicate-safe at the database boundary.
- Invoice numbering is serialized in MySQL because uniqueness is more important than optimistic convenience.
- Invoice queue creation also locks the local reservation row, so concurrent queue creation for the same reservation remains deterministic.
- Reservation persistence prefers narrow lookup queries over full local catalog scans when it resolves room and rate plan ids.
- Docker exists to prove reproducibility locally, not to pretend this repository is already a full deployment platform.

### Quick read

- the goal was clarity over abstraction
- the goal was stable integration behavior over cleverness
- the goal was a clean interview submission, not a fake framework

## Known Limits

- Live HotelSync command execution still depends on valid demo credentials in `.env`.
- `.env.example` intentionally keeps `HS_TOKEN` as a placeholder instead of shipping a concrete demo token value.
- Webhook authenticity is currently based on transport and payload validation because the brief does not define a signature contract.
- Invoice delivery currently runs inline with queue-state updates; a production version would split delivery into a worker path.
- Logging is plain text for the interview scope; correlation ids already exist, and a production service would add structured logs on top of them.
- Retry behavior is currently bounded by one shared invoice retry budget; a production version would classify transient and permanent delivery failures differently.
- Reservation import currently processes the requested range directly; a production version would add chunking, checkpoints, and shorter transaction scopes for larger sync windows.

## What the service does

### Catalog sync

- logs in to HotelSync
- fetches boards
- fetches rooms
- fetches pricing plans
- persists rooms with `HS-{ROOM_ID}-{slug_room_name}`
- persists rate plans with `RP-{RATE_PLAN_ID}-{meal_plan}`

Note:
The task mentions rooms and rate plans, but the real API exposes meal plan semantics through `boards`. That is why the catalog sync also pulls boards.

### Reservation sync

- imports reservations by date range
- imports one reservation by id
- computes canonical payload hashes
- avoids duplicate writes when the hash is unchanged
- keeps reservation rows even when status becomes canceled
- replaces child room and rate plan rows on payload changes
- writes audit rows for imported, updated, canceled, invoice, and webhook events

### Invoice flow

- builds invoice payloads from stored reservation payloads
- generates concurrency-safe numbers through `invoice_counters`
- queues invoices in `invoice_queue`
- retries failed delivery attempts up to the configured limit

Note:
The task does not define a real BridgeOne invoice endpoint. Delivery is therefore implemented as a configurable HTTP target through `BRIDGEONE_INVOICE_ENDPOINT`.

### Webhook flow

- accepts JSON webhooks only
- requires `POST`
- rejects oversized payloads
- hashes payloads for deduplication when event id is missing
- stores every event in `webhook_events`
- reuses reservation sync logic instead of duplicating update code

## Design decisions

- `app/bootstrap/bootstrap.php` loads only technical runtime pieces. Business files are loaded by feature bootstraps such as [`features/catalog/catalog_bootstrap.php`](features/catalog/catalog_bootstrap.php) and [`features/webhooks/webhooks_bootstrap.php`](features/webhooks/webhooks_bootstrap.php). That keeps `app/` technical and keeps business composition close to each feature.
- The project stays procedural on purpose. The task explicitly asks for procedural PHP and forbids frameworks that abstract HTTP or DB layers, so the code leans into a feature-first DSL instead of pretending to be a mini framework.
- Catalog sync reads `boards` before pricing plans because the upstream API exposes meal-plan semantics there. That is why local rate-plan codes use `RP-{RATE_PLAN_ID}-{meal_plan}` even though the task itself only names rate plans.
- Webhook dedupe prefers the upstream event id when it exists and falls back to a canonical payload hash when it does not. That makes duplicate handling stable across both rich and minimal webhook payloads.
- Webhook inbox persistence uses a DB-backed insert-or-reuse pattern on the unique dedupe key, so two near-simultaneous deliveries of the same event do not crash the endpoint or create duplicate rows.
- Invoice numbering is locked in MySQL through `invoice_counters` plus `SELECT ... FOR UPDATE`, because uniqueness matters more than optimistic convenience in this flow.
- Invoice queue creation also locks the local reservation row before queue insertion, so two concurrent invoice commands for the same reservation settle into `queued` plus `existing` instead of a unique-key collision.
- Docker is included as a reproducible local verification tool, not as a final production deployment manifest.
- The repository does not add Redis or a general cache layer because this service is correctness-first and the database is already the source of truth for idempotency, queue state, and auditability.
- Reservation persistence now loads only the room ids and rate plan ids referenced by the current payload instead of reading the full local catalog on every sync.

## Data integrity guarantees

The local schema is designed so the database makes important mistakes impossible:

- [`setup/database/create_schema.sql`](setup/database/create_schema.sql) locks rooms by `external_room_type_id` and rate plans by `external_rate_plan_id`
- reservations are unique by both `external_reservation_id` and `lock_id`
- invoice queue rows are unique by both `reservation_id` and `invoice_number`
- webhook inbox rows are unique by `dedupe_key`
- helper indexes support reservation-centric audit history, queue status scans, and webhook history reads
- reservation child rows are replaced inside one sync flow, which keeps room and rate-plan snapshots aligned with the latest payload

In practice, that means the source of truth is split cleanly:

- HotelSync payloads are the source of truth for reservation state
- the local database is the source of truth for idempotency, auditability, queue state, and stable local mappings

## Security notes

The code intentionally covers the interview-grade security basics that matter in integration work:

- prepared statements everywhere through `mysqli`
- hash-based idempotency for reservation updates
- webhook method and content-type validation
- webhook size limit to reduce noisy abuse vectors
- HTTPS enforcement for invoice delivery by default
- correlation ids carried through CLI entrypoints, webhook requests, logs, and outbound HTTP headers
- no raw secrets written into logs

This is not full production hardening yet. If this were a real service, I would add:

- webhook signature verification
- rate limiting
- structured logs
- queue worker separation
- dead-letter handling for repeated invoice failures
- stronger outbound HTTP allow-listing
- secret rotation and dedicated webhook signature verification, if the upstream sender supports it

## Known limitations

- The Docker proof path is fully reproducible locally, but the real HotelSync command flows still require valid demo credentials in `.env`
- Invoice delivery is modeled as a synchronous HTTP call plus queue state updates; a production version would move delivery and retries into a worker lane
- Webhook authenticity currently relies on transport and payload validation because the upstream brief does not define a signature scheme
- Logs are intentionally plain text for the interview scope; correlation ids already exist, and a production service would add structured logs on top of them
- `.env.example` intentionally keeps `HS_TOKEN` as a placeholder so the repository does not ship with a concrete demo token value
- Invoice retries currently follow one bounded retry budget; a production version would classify transient and permanent errors differently
- Reservation import currently processes the requested range directly; a production version would add chunking/checkpointing and shorter transaction scopes for larger date windows

## How I would extend this

- move invoice delivery into a dedicated worker loop with retry scheduling and dead-letter handling
- add webhook signature verification when the upstream sender provides a signing contract
- enforce an outbound allow-list for invoice destinations
- add structured logs on top of the existing correlation ids across CLI, webhook, and outbound invoice flows
- classify outbound retry behavior by error type, so timeouts and 5xx responses retry while permanent 4xx validation failures fail fast
- add chunked reservation imports with smaller transaction scopes and resumable checkpoints for larger sync windows
- expose a lightweight health/probe surface for operators once the service becomes long-running

## Assumptions

- The HotelSync Postman collection is the source of truth for endpoint shapes.
- The Docker setup is meant for reproducible local verification, not as a final production deployment manifest.
- `pricing plans` use `boards` for readable meal plan naming.
- Reservation ids are numeric in CLI and webhook flows.
- Re-running `generate_invoice.php` for the same reservation reuses the existing queue row instead of minting a second invoice.
- Webhook payloads may contain either:
  - the full reservation payload
  - or only a reservation id, in which case the service fetches the latest reservation from HotelSync

<?php

declare(strict_types=1);

/**
 * Parses long CLI flags into a small option array without pulling a command framework into the project.
 *
 * @param array<int, string> $argv Raw CLI argv array.
 *
 * @return array<string, mixed>
 */
function cli_options(array $argv): array
{
    $options = [];

    foreach ($argv as $argument) {
        if (!str_starts_with(haystack: $argument, needle: '--')) {
            continue;
        }

        $argument = substr(string: $argument, offset: 2);
        $parts = explode(separator: '=', string: $argument, limit: 2);
        $key = $parts[0];
        $value = $parts[1] ?? true;
        $options[$key] = $value;
    }

    return $options;
}

/**
 * Reads one required CLI option and throws a precise message when the flag is missing.
 *
 * @param array<string, mixed> $options Parsed CLI options.
 * @param string $key Required option name without the leading dashes.
 *
 * @return string
 *
 * @throws InvalidArgumentException When the option is missing.
 */
function cli_require_option(array $options, string $key): string
{
    if (!array_key_exists(key: $key, array: $options) || $options[$key] === '' || $options[$key] === true) {
        throw new InvalidArgumentException(message: 'Missing required option --' . $key);
    }

    return (string) $options[$key];
}

<?php

declare(strict_types=1);

/**
 * Opens one mysqli connection per request and configures it to fail with exceptions instead of silent warnings.
 *
 * @return mysqli
 */
function db_connection(): mysqli
{
    static $connection = null;

    if ($connection instanceof mysqli) {
        return $connection;
    }

    mysqli_report(flags: MYSQLI_REPORT_ERROR | MYSQLI_REPORT_STRICT);

    $connection = new mysqli(
        hostname: (string) config_get(key: 'database.host'),
        username: (string) config_get(key: 'database.username'),
        password: (string) config_get(key: 'database.password'),
        database: (string) config_get(key: 'database.database'),
        port: (int) config_get(key: 'database.port')
    );

    $connection->set_charset(charset: (string) config_get(key: 'database.charset', default: 'utf8mb4'));

    return $connection;
}

/**
 * Prepares, binds, and executes one SQL statement so the call site stays focused on intent, not plumbing.
 *
 * @param mysqli $db Open mysqli connection.
 * @param string $sql SQL statement with placeholders.
 * @param string $types mysqli bind_param type string.
 * @param array<int, mixed> $params Parameters bound to the statement.
 *
 * @return mysqli_stmt
 */
function db_statement(mysqli $db, string $sql, string $types = '', array $params = []): mysqli_stmt
{
    $statement = $db->prepare(query: $sql);

    if ($statement === false) {
        throw new RuntimeException(message: 'Unable to prepare SQL statement.');
    }

    if ($types !== '' && $params !== []) {
        $bindValues = [$types];

        foreach (array_keys(array: $params) as $index) {
            $bindValues[] = &$params[$index];
        }

        call_user_func_array(callback: [$statement, 'bind_param'], args: $bindValues);
    }

    $statement->execute();

    return $statement;
}

/**
 * Fetches every row as an associative array for read paths that need the full result set.
 *
 * @param mysqli $db Open mysqli connection.
 * @param string $sql SQL statement with placeholders.
 * @param string $types mysqli bind_param type string.
 * @param array<int, mixed> $params Parameters bound to the statement.
 *
 * @return array<int, array<string, mixed>>
 */
function db_fetch_all(mysqli $db, string $sql, string $types = '', array $params = []): array
{
    $statement = db_statement(db: $db, sql: $sql, types: $types, params: $params);
    $result = $statement->get_result();
    $rows = $result instanceof mysqli_result ? $result->fetch_all(mode: MYSQLI_ASSOC) : [];
    $statement->close();

    return $rows;
}

/**
 * Fetches the first matching row and returns null when the record simply does not exist.
 *
 * @param mysqli $db Open mysqli connection.
 * @param string $sql SQL statement with placeholders.
 * @param string $types mysqli bind_param type string.
 * @param array<int, mixed> $params Parameters bound to the statement.
 *
 * @return array<string, mixed>|null
 */
function db_fetch_one(mysqli $db, string $sql, string $types = '', array $params = []): array|null
{
    $rows = db_fetch_all(db: $db, sql: $sql, types: $types, params: $params);

    return $rows[0] ?? null;
}

/**
 * Runs a write query and returns the affected row count for callers that care about write impact.
 *
 * @param mysqli $db Open mysqli connection.
 * @param string $sql SQL statement with placeholders.
 * @param string $types mysqli bind_param type string.
 * @param array<int, mixed> $params Parameters bound to the statement.
 *
 * @return int
 */
function db_execute(mysqli $db, string $sql, string $types = '', array $params = []): int
{
    $statement = db_statement(db: $db, sql: $sql, types: $types, params: $params);
    $affectedRows = $statement->affected_rows;
    $statement->close();

    return $affectedRows;
}

<?php

declare(strict_types=1);

/**
 * Overrides the JSON POST transport, which is useful for deterministic tests around upstream failures.
 *
 * @param callable|null $transport Custom transport callback or null to restore the default cURL transport.
 *
 * @return void
 */
function http_override_post_json_transport(callable|null $transport): void
{
    if ($transport === null) {
        unset($GLOBALS['__http_post_json_transport']);

        return;
    }

    $GLOBALS['__http_post_json_transport'] = $transport;
}

/**
 * Adds transport-level headers that every outbound JSON call should carry, such as the active correlation id.
 *
 * @param array<string, string> $headers Caller-provided headers.
 *
 * @return array<string, string>
 */
function http_enrich_outbound_json_headers(array $headers): array
{
    $normalizedHeaderNames = array_change_key_case(array: $headers, case: CASE_LOWER);

    if (app_get_correlation_id() !== '' && !array_key_exists(key: 'x-correlation-id', array: $normalizedHeaderNames)) {
        $headers['X-Correlation-Id'] = app_get_correlation_id();
    }

    return $headers;
}

/**
 * Sends a JSON POST request and returns both raw and decoded data so callers can decide how strict to be.
 *
 * @param string $url Target URL.
 * @param array<string, mixed> $payload JSON payload.
 * @param array<string, string> $headers Extra request headers.
 * @param int|null $timeoutSeconds Request timeout in seconds.
 *
 * @return array{status_code:int,headers:array<string,string>,body:string,json:array<mixed>|null}
 *
 * @throws RuntimeException When cURL cannot run the request.
 */
function http_post_json(string $url, array $payload, array $headers = [], int|null $timeoutSeconds = null): array
{
    $headers = http_enrich_outbound_json_headers(headers: $headers);

    if (isset($GLOBALS['__http_post_json_transport']) && is_callable(value: $GLOBALS['__http_post_json_transport'])) {
        $response = $GLOBALS['__http_post_json_transport']($url, $payload, $headers, $timeoutSeconds);

        if (!is_array(value: $response)) {
            throw new RuntimeException(message: 'Custom HTTP transport must return an array response.');
        }

        return $response;
    }

    $ch = curl_init(url: $url);

    if ($ch === false) {
        throw new RuntimeException(message: 'Unable to initialize cURL.');
    }

    $requestHeaders = ['Content-Type: application/json'];

    foreach ($headers as $key => $value) {
        $requestHeaders[] = $key . ': ' . $value;
    }

    $responseHeaders = [];

    curl_setopt_array(handle: $ch, options: [
        CURLOPT_POST => true,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_FOLLOWLOCATION => false,
        CURLOPT_HTTPHEADER => $requestHeaders,
        CURLOPT_POSTFIELDS => app_json_encode(value: $payload),
        CURLOPT_CONNECTTIMEOUT => max(1, min($timeoutSeconds ?? 30, 10)),
        CURLOPT_TIMEOUT => $timeoutSeconds ?? 30,
        CURLOPT_HEADERFUNCTION => static function ($curl, string $headerLine) use (&$responseHeaders): int {
            $length = strlen(string: $headerLine);
            $headerLine = trim(string: $headerLine);

            if ($headerLine === '' || !str_contains(haystack: $headerLine, needle: ':')) {
                return $length;
            }

            [$name, $value] = explode(separator: ':', string: $headerLine, limit: 2);
            $responseHeaders[strtolower(string: trim(string: $name))] = trim(string: $value);

            return $length;
        },
    ]);

    $body = curl_exec(handle: $ch);

    if ($body === false) {
        $error = curl_error(handle: $ch);
        curl_close(handle: $ch);
        throw new RuntimeException(message: 'HTTP request failed: ' . $error);
    }

    $statusCode = (int) curl_getinfo(handle: $ch, option: CURLINFO_RESPONSE_CODE);
    curl_close(handle: $ch);

    $json = null;
    $trimmedBody = trim(string: $body);

    if ($trimmedBody !== '' && ($trimmedBody[0] === '{' || $trimmedBody[0] === '[')) {
        try {
            $json = app_json_decode(value: $trimmedBody);
        } catch (RuntimeException) {
            $json = null;
        }
    }

    return [
        'status_code' => $statusCode,
        'headers' => $responseHeaders,
        'body' => $body,
        'json' => $json,
    ];
}

/**

function hotelsync_assert_login_config(): void

    $username = (string) config_get(key: 'app.hotelsync.username');
    $password = (string) config_get(key: 'app.hotelsync.password');

/**
 * Signs in to HotelSync and extracts the session pieces this integration needs for every later request.
 *
 * @return array{token:string,property_id:string,api_key:string,login_payload:array<string,mixed>}
 *
 * @throws RuntimeException When login fails or the session payload is incomplete.
 */
function hotelsync_login(): array
{
    hotelsync_assert_login_config();

    $response = http_post_json(
        url: hotelsync_base_url() . '/user/auth/login',
        payload: [
            'token' => (string) config_get(key: 'app.hotelsync.token'),
            'username' => (string) config_get(key: 'app.hotelsync.username'),
            'password' => (string) config_get(key: 'app.hotelsync.password'),
            'remember' => (int) config_get(key: 'app.hotelsync.remember', default: 0),
        ],
        headers: [],
        timeoutSeconds: (int) config_get(key: 'app.hotelsync.timeout_seconds', default: 30)
    );

    if ($response['status_code'] < 200 || $response['status_code'] >= 300) {
        throw new RuntimeException(message: 'HotelSync login failed with status ' . $response['status_code'] . '.');
    }

    if (!is_array(value: $response['json'])) {
        throw new RuntimeException(message: 'HotelSync login did not return valid JSON.');
    }

    $loginPayload = $response['json'];

    // The upstream login payload is considered usable only after the session scalars are normalized.
    $propertyId = app_optional_scalar_to_string(
        value: app_first_present(
            payload: $loginPayload,
            paths: ['id_properties', 'data.id_properties', 'property.id_properties', 'properties.0.id_properties'],
            default: (string) config_get(key: 'app.hotelsync.property_id', default: '')
        ),
        default: ''
    );
    $apiKey = app_optional_scalar_to_string(
        value: app_first_present(
            payload: $loginPayload,
            paths: ['key', 'data.key', 'session.key'],
            default: (string) config_get(key: 'app.hotelsync.api_key', default: '')
        ),
        default: ''
    );

    if ($propertyId === '' || $apiKey === '') {
        throw new RuntimeException(
            message: 'HotelSync property_id/api_key are missing. Configure HS_PROPERTY_ID and HS_API_KEY if login response does not expose them.'
        );
    }

    return [
        'token' => (string) config_get(key: 'app.hotelsync.token'),
        'property_id' => $propertyId,
        'api_key' => $apiKey,
        'login_payload' => $loginPayload,
    ];
}

/**
 * Sends one HotelSync POST request and enforces JSON responses so the rest of the code can stay predictable.
 *
 * @param string $path Relative API path or full URL.
 * @param array<string, mixed> $payload Request payload.
 * @param array{
 *     token:string,
 *     property_id:string,
 *     api_key:string,
 *     login_payload?:array<string, mixed>
 * }|null $session Auth session returned by hotelsync_login().
 *
 * @return array<string, mixed>
 *
 * @throws RuntimeException When the request fails or the response is not JSON.
 */
function hotelsync_post(string $path, array $payload, array|null $session = null): array
{
    $url = str_starts_with(haystack: $path, needle: 'http') ? $path : hotelsync_base_url() . '/' . ltrim(string: $path, characters: '/');

    if ($session !== null) {
        $payload = array_merge(
            [
                'token' => $session['token'],
                'id_properties' => $session['property_id'],
                'key' => $session['api_key'],
            ],

        );
    }

    $response = http_post_json(
        url: $url,
        payload: $payload,
        headers: [],
        timeoutSeconds: (int) config_get(key: 'app.hotelsync.timeout_seconds', default: 30)
    );

    if ($response['status_code'] < 200 || $response['status_code'] >= 300) {
        throw new RuntimeException(

        );
    }

    if (!is_array(value: $response['json'])) {
        throw new RuntimeException(message: 'HotelSync response was not valid JSON for ' . $url);
    }

    return $response['json'];
}

/**
 * Extracts one list of records from either a direct JSON list response or a wrapper object with a nested list key.
 *
 * @param array<string|int, mixed> $response Raw HotelSync JSON response.
 * @param array<int, string> $candidateKeys Preferred wrapper keys to inspect first.
 *
 * @return array<int, array<string, mixed>>
 */
function hotelsync_extract_list(array $response, array $candidateKeys): array
{
    if (array_is_list(array: $response)) {
        return array_values(
            array_filter(
                $response,
                static fn (mixed $item): bool => is_array(value: $item)
            )
        );
    }

    foreach ($candidateKeys as $candidateKey) {
        $candidate = app_first_present(payload: $response, paths: [$candidateKey], default: null);

        if (!is_array(value: $candidate)) {
            continue;
        }

        return array_values(
            array_filter(
                $candidate,
                static fn (mixed $item): bool => is_array(value: $item)
            )
        );
    }

    return array_values(
        array_filter(
            $response,
            static fn (mixed $item): bool => is_array(value: $item)
        )
    );
}

/**
 * Appends one flat log line with the key context fields this integration needs during debugging.
 * The active correlation id is added automatically when an entrypoint initialized one.
 *
 * @param string $eventType Short machine-friendly event name.
 * @param string $message Human-readable log message.
 * @param string|null $reservationId Local reservation id when available.
 * @param string|null $externalId External reservation id or any other external reference.
 *
 * @return void
 */
function logger_write(string $eventType, string $message, string|null $reservationId = null, string|null $externalId = null): void

    app_ensure_directory(path: dirname(path: $logFile));

        '[' . app_now() . ']',
        strtoupper(string: $eventType),
        $message,
    ];

    if ($reservationId !== null && $reservationId !== '') {
        $parts[] = 'reservation_id=' . $reservationId;

        $parts[] = 'external_id=' . $externalId;

<?php

/**
 * Returns the current application timestamp in the same format used across SQL writes and logs.
 *
 * @return string
 */
function app_now(): string
{
    return date(format: 'Y-m-d H:i:s');
}

/**
 * Turns a human label into a compact ASCII slug that is safe inside local integration codes.
 *
 * @param string $value Human-readable value that will be slugged.
 *
 * @return string
 */
function app_slugify(string $value): string
{
    $ascii = iconv(from_encoding: 'UTF-8', to_encoding: 'ASCII//TRANSLIT//IGNORE', string: $value);
    $ascii = $ascii === false ? $value : $ascii;
    $slug = strtolower(string: trim(string: (string) preg_replace(pattern: '/[^A-Za-z0-9]+/', replacement: '-', subject: $ascii), characters: '-'));

    return $slug !== '' ? $slug : 'n-a';
}

/**
 * Encodes JSON with stable flags so payloads stay readable and hashes stay predictable.
 *
 * @param mixed $value Data structure that should become JSON.
 *
 * @return string
 *
 * @throws RuntimeException When JSON encoding fails.
 */
function app_json_encode(mixed $value): string
{
    $encoded = json_encode(
        value: $value,
        flags: JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES | JSON_PRESERVE_ZERO_FRACTION
    );

    if ($encoded === false) {
        throw new RuntimeException(message: 'Unable to encode JSON payload.');
    }

    return $encoded;
}

/**
 * Decodes a JSON string and raises a clear exception instead of letting broken payloads slide by.
 *
 * @param string $value Raw JSON string.
 *
 * @return mixed
 *
 * @throws RuntimeException When JSON decoding fails.
 */
function app_json_decode(string $value): mixed
{
    $decoded = json_decode(json: $value, associative: true);

    if (json_last_error() !== JSON_ERROR_NONE) {
        throw new RuntimeException(message: 'Unable to decode JSON payload: ' . json_last_error_msg());
    }

    return $decoded;
}

/**
 * Detects whether an array behaves like an object map instead of a positional list.
 *
 * @param array<mixed> $value Array to inspect.
 *
 * @return bool
 */
function app_is_assoc_array(array $value): bool
{
    return !array_is_list(array: $value);
}

/**
 * Sorts associative structures recursively so the same payload produces the same hash every time.
 *
 * @param mixed $value Payload fragment to normalize.
 *
 * @return mixed
 */
function app_normalize_for_hash(mixed $value): mixed
{
    if (!is_array(value: $value)) {
        return $value;
    }

    if (app_is_assoc_array(value: $value)) {
        ksort(array: $value);
    }

    foreach ($value as $key => $item) {
        $value[$key] = app_normalize_for_hash(value: $item);
    }

    return $value;
}

/**
 * Builds the canonical SHA-256 hash used for idempotency checks and audit comparisons.
 *
 * @param array<string, mixed> $payload Payload that should be hashed.
 *
 * @return string
 */
function app_payload_hash(array $payload): string
{
    return hash(algo: 'sha256', data: app_json_encode(value: app_normalize_for_hash(value: $payload)));
}

/**
 * Creates a directory tree on demand and fails loudly when the filesystem refuses it.
 *
 * @param string $path Absolute or relative directory path.
 *
 * @return void
 *
 * @throws RuntimeException When the directory cannot be created.

function app_ensure_directory(string $path): void
{
    if (is_dir(filename: $path)) {
        return;

    if (!mkdir(directory: $path, permissions: 0775, recursive: true) && !is_dir(filename: $path)) {

/**

 *
 * @return string
 */
function app_generate_correlation_id(): string
{
    try {
        $entropy = bin2hex(string: random_bytes(length: 8));
    } catch (Throwable) {
        $entropy = substr(string: hash(algo: 'sha256', data: uniqid(prefix: 'corr', more_entropy: true)), offset: 0, length: 16);
    }

    return 'corr-' . $entropy;

/**
 * Starts or replaces the current correlation context with a sanitized preferred value or a generated id.
 *
 * @param mixed $preferredCorrelationId Optional upstream or CLI-provided correlation id.
 *
 * @return string
 */
function app_start_correlation_context(mixed $preferredCorrelationId = null): string
{
    $preferredCorrelationId = app_optional_scalar_to_string(value: $preferredCorrelationId, default: '');

    if ($preferredCorrelationId !== '') {
        $preferredCorrelationId = (string) preg_replace(
            pattern: '/[^A-Za-z0-9._-]+/',
            replacement: '-',
            subject: $preferredCorrelationId
        );
        $preferredCorrelationId = trim(string: $preferredCorrelationId, characters: '-.');
    }

    $correlationId = $preferredCorrelationId !== ''
        ? substr(string: $preferredCorrelationId, offset: 0, length: 96)
        : app_generate_correlation_id();

    $GLOBALS['__app_correlation_id'] = $correlationId;

    return $correlationId;
}

    return is_string(value: $currentCorrelationId) ? $currentCorrelationId : '';
}

/**
 * Tries several nested paths and returns the first non-empty value, which is handy for messy APIs.
 *
 * @param array<string, mixed> $payload Source payload.
 * @param array<int, string> $paths Candidate dot paths.
 * @param mixed $default Value returned when none of the paths is present.
 *
 * @return mixed
 */
function app_first_present(array $payload, array $paths, mixed $default = null): mixed
{
    foreach ($paths as $path) {
        $segments = explode(separator: '.', string: $path);
        $value = $payload;
        $found = true;

        foreach ($segments as $segment) {
            if (!is_array(value: $value) || !array_key_exists(key: $segment, array: $value)) {
                $found = false;
                break;
            }

            $value = $value[$segment];
        }

        if ($found && $value !== null && $value !== '') {
            return $value;
        }
    }

    return $default;
}

/**
 * Adds up every children_* field so room and reservation totals do not need seven manual checks.
 *
 * @param array<string, mixed> $payload Payload fragment with children counters.
 *
 * @return int
 */
function app_sum_children(array $payload): int
{
    $total = 0;

    for ($index = 1; $index <= 7; $index++) {
        $total += (int) ($payload['children_' . $index] ?? 0);

}

/**
 * Converts an optional scalar-like value into a trimmed string and falls back when the input shape is not usable.
 *
 * @param mixed $value Raw upstream value.
 * @param string $default Value returned for null, empty strings, arrays, or objects.
 *
 * @return string
 */
function app_optional_scalar_to_string(mixed $value, string $default = ''): string
{
    if ($value === null || is_array(value: $value) || is_object(value: $value) || is_resource(value: $value)) {
        return $default;
    }

    $normalized = trim(string: (string) $value);

    return $normalized !== '' ? $normalized : $default;
}

/**
 * Requires one upstream value to be present and non-empty before the integration can safely continue.
 *
 * @param mixed $value Raw upstream value.
 * @param string $label Human-readable field label used in the exception message.
 *
 * @return string
 *
 * @throws RuntimeException When the value is empty.
 */
function app_require_non_empty_string(mixed $value, string $label): string
{
    if (is_array(value: $value) || is_object(value: $value) || is_resource(value: $value)) {
        throw new RuntimeException(message: 'Required value has invalid type for ' . $label . '.');
    }

    $normalized = trim(string: (string) $value);

    if ($normalized === '') {
        throw new RuntimeException(message: 'Required value is missing: ' . $label . '.');
    }

    return $normalized;
}

/**
 * Requires one upstream date string to match YYYY-MM-DD before it reaches the local reservation schema.
 *
 * @param mixed $value Raw upstream date value.
 * @param string $label Human-readable field label used in the exception message.
 *
 * @return string
 *
 * @throws RuntimeException When the date is missing or malformed.
 */
function app_require_iso_date_string(mixed $value, string $label): string
{
    $normalized = app_require_non_empty_string(value: $value, label: $label);
    $date = DateTimeImmutable::createFromFormat(format: 'Y-m-d', datetime: $normalized);

    if ($date === false || $date->format(format: 'Y-m-d') !== $normalized) {
        throw new RuntimeException(message: 'Required date is invalid for ' . $label . ': ' . $normalized . '.');
    }

    return $normalized;
}

/**
 * Checks whether a URL is explicitly HTTPS before sensitive payloads are sent to it.
 *
 * @param string $value URL to inspect.
 *
 * @return bool
 */
function app_is_https_url(string $value): bool
{
    if ($value === '') {
        return false;
    }

    $parts = parse_url(url: $value);

    return is_array(value: $parts) && strtolower(string: (string) ($parts['scheme'] ?? '')) === 'https';
}

/**
 * Treats JSON content types with optional charset parameters as valid webhook input.
 *
 * @param string|null $contentType Raw Content-Type header value.
 *
 * @return bool
 */
function app_content_type_is_json(string|null $contentType): bool
{
    if ($contentType === null || $contentType === '') {
        return false;
    }

    $mimeType = strtolower(string: trim(string: explode(separator: ';', string: $contentType, limit: 2)[0]));

    return $mimeType === 'application/json';
}

=== features/catalog/sync_catalog_from_hotelsync.php ===
<?php

declare(strict_types=1);

/**
 * Runs the full catalog sync flow: sign in, fetch boards, fetch rooms, fetch rate plans, then persist everything.
 *
 * @return array{rooms:int,rate_plans:int}
 *
 * @throws Throwable When one of the sync steps fails.
 */
function sync_catalog_from_hotelsync(): array
{
    $db = db_connection();
    $session = hotelsync_login();
    $boards = fetch_boards_from_hotelsync(session: $session);
    $rooms = fetch_rooms_from_hotelsync(session: $session);
    $ratePlans = fetch_rate_plans_from_hotelsync(session: $session);
    $boardNamesById = transform_boards_payload_to_board_names_by_id(boards: $boards);

    $db->begin_transaction();

    try {
        foreach ($rooms as $room) {
            if (is_array(value: $room)) {
                upsert_room_catalog_record(db: $db, row: transform_room_payload_to_catalog_record(room: $room));
            }
        }

        foreach ($ratePlans as $ratePlan) {
            if (is_array(value: $ratePlan)) {
                upsert_rate_plan_catalog_record(db: $db, row: transform_rate_plan_payload_to_catalog_record(ratePlan: $ratePlan, boardNamesById: $boardNamesById));
            }
        }

        $db->commit();
    } catch (Throwable $exception) {
        $db->rollback();
        logger_write(eventType: 'catalog_error', message: $exception->getMessage());
        throw $exception;
    }

    logger_write(
        eventType: 'catalog_sync',
        message: 'Catalog sync completed.',
        reservationId: null,
        externalId: 'rooms=' . count(value: $rooms) . ';rate_plans=' . count(value: $ratePlans)
    );

    return [
        'rooms' => count(value: $rooms),
        'rate_plans' => count(value: $ratePlans),
    ];
}

=== features/invoices/delivery/deliver_invoice_from_queue.php ===
<?php

declare(strict_types=1);

/**
 * Sends one queued invoice to the configured BridgeOne endpoint and updates retry state on failure.
 *
 * @param int $invoiceQueueId Local invoice queue id.
 *
 * @return array<string, mixed>
 *
 * @throws Throwable When delivery fails.
 */
function deliver_invoice_from_queue(int $invoiceQueueId): array
{
    $db = db_connection();
    $invoiceQueue = find_invoice_queue_item_by_id(db: $db, invoiceQueueId: $invoiceQueueId);

    if ($invoiceQueue === null) {
        throw new RuntimeException(message: 'Invoice queue item not found.');
    }

    if ((string) $invoiceQueue['status'] === 'sent') {
        return $invoiceQueue;
    }

    $endpoint = (string) config_get(key: 'app.invoice.endpoint', default: '');

    if ($endpoint === '') {
        return $invoiceQueue;
    }

    $reservation = find_local_reservation_by_id(db: $db, reservationId: (int) $invoiceQueue['reservation_id']);

    if ($reservation === null) {
        throw new RuntimeException(message: 'Reservation not found for invoice queue item.');
    }

    if (!mark_invoice_queue_item_as_processing(db: $db, invoiceQueueId: $invoiceQueueId)) {
        $currentQueueState = find_invoice_queue_item_by_id(db: $db, invoiceQueueId: $invoiceQueueId);

        if ($currentQueueState === null) {
            throw new RuntimeException(message: 'Invoice queue item disappeared before delivery could start.');
        }

        return $currentQueueState;
    }

    try {
        if ((int) config_get(key: 'app.invoice.require_https', default: 1) === 1 && !app_is_https_url(value: $endpoint)) {
            throw new RuntimeException(message: 'BridgeOne invoice endpoint must use HTTPS.');
        }

        $payload = app_json_decode(value: (string) $invoiceQueue['payload_json']);

        if (!is_array(value: $payload)) {
            throw new RuntimeException(message: 'Invoice queue payload must decode into a JSON object.');
        }

        $response = http_post_json(
            url: $endpoint,
            payload: $payload,
            headers: [],
            timeoutSeconds: (int) config_get(key: 'app.invoice.timeout_seconds', default: 15)
        );

        if ($response['status_code'] < 200 || $response['status_code'] >= 300) {
            throw new RuntimeException(message: 'Invoice delivery failed with status ' . $response['status_code'] . '.');
        }

        mark_invoice_queue_item_as_sent(db: $db, invoiceQueueId: $invoiceQueueId);

        write_reservation_audit_log(
            db: $db,
            reservationId: (int) $reservation['id'],
            externalReservationId: (string) $reservation['external_reservation_id'],
            eventType: 'invoice_sent',
            oldPayloadHash: null,
            newPayloadHash: null,
            message: 'Invoice delivered to BridgeOne endpoint.',
            payload: $payload
        );
    } catch (Throwable $exception) {
        try {
            $failedPayload = app_json_decode(value: (string) $invoiceQueue['payload_json']);
        } catch (Throwable) {
            $failedPayload = [
                'raw_payload_json' => (string) $invoiceQueue['payload_json'],
            ];
        }

        mark_invoice_queue_item_for_retry(
            db: $db,
            invoiceQueue: $invoiceQueue,
            errorMessage: $exception->getMessage(),
            maxRetries: (int) config_get(key: 'app.invoice.max_retries', default: 5)
        );

        write_reservation_audit_log(
            db: $db,
            reservationId: (int) $reservation['id'],
            externalReservationId: (string) $reservation['external_reservation_id'],
            eventType: 'invoice_failed',
            oldPayloadHash: null,
            newPayloadHash: null,
            message: $exception->getMessage(),
            payload: is_array(value: $failedPayload) ? $failedPayload : [
                'raw_payload_json' => (string) $invoiceQueue['payload_json'],
            ]
        );

        logger_write(
            eventType: 'invoice_failed',
            message: $exception->getMessage(),
            reservationId: (string) $reservation['id'],
            externalId: (string) $reservation['external_reservation_id']
        );

        throw $exception;
    }

    logger_write(eventType: 'invoice_sent', message: 'Invoice delivered.', reservationId: (string) $reservation['id'], externalId: (string) $reservation['external_reservation_id']);

    $updatedInvoiceQueue = find_invoice_queue_item_by_id(db: $db, invoiceQueueId: $invoiceQueueId);

    if ($updatedInvoiceQueue === null) {
        throw new RuntimeException(message: 'Invoice queue item disappeared after delivery.');
    }

    return $updatedInvoiceQueue;
}

=== features/invoices/generate_and_queue_invoice_for_reservation.php ===
<?php

declare(strict_types=1);

/**
 * Creates or reuses the queued invoice for one external reservation id and optionally kicks off delivery.
 *
 * @param string $externalReservationId HotelSync reservation id.
 *
 * @return array{
 *     action:string,
 *     invoice_number:string,
 *     queue_id:int,
 *     status?:string
 * }
 *
 * @throws Throwable When reservation loading, queue creation, or delivery bootstrap fails.
 */
function generate_and_queue_invoice_for_reservation(string $externalReservationId): array
{
    $reservation = load_or_sync_reservation_by_external_id(externalReservationId: $externalReservationId);
    $db = db_connection();
    $db->begin_transaction();

    try {
        // Invoice creation serializes on the reservation row so two concurrent commands cannot mint two numbers.
        lock_local_reservation_by_id(db: $db, reservationId: (int) $reservation['id']);
        $existingInvoice = find_invoice_queue_item_by_reservation_id(db: $db, reservationId: (int) $reservation['id'], lock: true);

        if ($existingInvoice !== null) {
            $db->commit();
            logger_write(
                eventType: 'invoice_exists',
                message: 'Invoice already queued for reservation.',
                reservationId: (string) $reservation['id'],
                externalId: $externalReservationId
            );

            return [
                'action' => 'existing',
                'invoice_number' => (string) $existingInvoice['invoice_number'],
                'queue_id' => (int) $existingInvoice['id'],
                'status' => (string) $existingInvoice['status'],
            ];
        }

        $year = (int) date(format: 'Y', timestamp: strtotime(datetime: (string) $reservation['arrival_date']));
        $invoiceNumber = reserve_next_invoice_number(db: $db, year: $year);
        $payload = transform_local_reservation_to_invoice_payload(reservation: $reservation, invoiceNumber: $invoiceNumber);
        $queueId = save_invoice_queue_item(db: $db, reservationId: (int) $reservation['id'], invoiceNumber: $invoiceNumber, payload: $payload);

        write_reservation_audit_log(
            db: $db,
            reservationId: (int) $reservation['id'],
            externalReservationId: (string) $reservation['external_reservation_id'],
            eventType: 'invoice_created',
            oldPayloadHash: null,
            newPayloadHash: null,
            message: 'Invoice queued for delivery.',
            payload: $payload
        );

        $db->commit();
    } catch (Throwable $exception) {
        $db->rollback();
        logger_write(eventType: 'invoice_error', message: $exception->getMessage(), reservationId: (string) ($reservation['id'] ?? ''), externalId: $externalReservationId);
        throw $exception;
    }

    logger_write(eventType: 'invoice_queued', message: 'Invoice queued.', reservationId: (string) $reservation['id'], externalId: $externalReservationId);

    if ((string) config_get(key: 'app.invoice.endpoint', default: '') !== '') {
        deliver_invoice_from_queue(invoiceQueueId: $queueId);
    }

    return [
        'action' => 'queued',
        'invoice_number' => $invoiceNumber,
        'queue_id' => $queueId,
    ];
}

<?php

declare(strict_types=1);

/**
 * Builds the invoice line items from the raw reservation payload so room and extras stay visible in the queue.
 *
 * @param array<string, mixed> $payload Raw reservation payload.
 *
 * @return array<int, array<string, int|float|string>>
 */
function transform_reservation_payload_to_invoice_line_items(array $payload): array
{
    $lineItems = [];
    $rooms = is_array(value: ($payload['rooms'] ?? null))
        ? $payload['rooms']
        : [];
    $extras = is_array(value: ($payload['extras'] ?? null))
        ? $payload['extras']
        : [];

    foreach ($rooms as $room) {
        if (!is_array(value: $room)) {
            continue;
        }

        $totalAmount = (float) ($room['total_price'] ?? 0);
        $description = 'Room: ' . (string) ($room['name'] ?? 'Room');

        if (!empty($room['room_number'])) {
            $description .= ' #' . (string) $room['room_number'];
        }

        $lineItems[] = [
            'description' => $description,
            'quantity' => 1,
            'unit_amount' => $totalAmount,
            'total_amount' => $totalAmount,
        ];
    }

    foreach ($extras as $extra) {
        if (!is_array(value: $extra)) {
            continue;
        }

        $quantity = (int) ($extra['quantity'] ?? 1);
        $unitAmount = (float) ($extra['price_per_unit'] ?? 0);
        $totalAmount = (float) ($extra['total_price'] ?? ($unitAmount * $quantity));

        $lineItems[] = [
            'description' => 'Extra: ' . (string) ($extra['name'] ?? 'Extra'),
            'quantity' => $quantity,
            'unit_amount' => $unitAmount,
            'total_amount' => $totalAmount,
        ];
    }

    $cityTax = (float) ($payload['city_tax_price'] ?? 0);

    if ($cityTax > 0) {
        $lineItems[] = [
            'description' => 'City tax',
            'quantity' => 1,
            'unit_amount' => $cityTax,
            'total_amount' => $cityTax,
        ];
    }

    $customTax = (float) ($payload['custom_tax_price'] ?? 0);

    if ($customTax > 0) {
        $lineItems[] = [
            'description' => (string) ($payload['custom_tax_name'] ?? 'Custom tax'),
            'quantity' => 1,
            'unit_amount' => $customTax,
            'total_amount' => $customTax,
        ];
    }

    if ($lineItems === []) {
        $totalAmount = (float) ($payload['total_price'] ?? 0);
        $lineItems[] = [
            'description' => 'Reservation total',
            'quantity' => 1,
            'unit_amount' => $totalAmount,
            'total_amount' => $totalAmount,
        ];
    }

    return $lineItems;
}

<?php

declare(strict_types=1);

/**
 * Applies one reservation payload to the local model with hash-based idempotency and audit logging.
 *
 * @param array<string, mixed> $reservationPayload Raw reservation payload.
 * @param string $context Sync context used in logging and audit rows.
 * @param string $currency Fallback currency when the payload does not expose one.
 *
 * @return array{action:string,reservation_id:int,external_reservation_id:string}
 *
 * @throws Throwable When the reservation cannot be saved.
 */
function sync_reservation_payload_to_local_state(
    array $reservationPayload,
    string $context = 'manual',
    string $currency = 'EUR'
): array
{
    $db = db_connection();
    $mapped = transform_reservation_payload_to_local_record_set(reservation: $reservationPayload, currency: $currency);
    $externalReservationId = $mapped['reservation']['external_reservation_id'];
    $existing = find_local_reservation_by_external_id(db: $db, externalReservationId: $externalReservationId);
    $newPayloadHash = $mapped['reservation']['payload_hash'];

    if ($existing !== null && (string) $existing['payload_hash'] === $newPayloadHash) {
        touch_local_reservation_sync_time(db: $db, reservationId: (int) $existing['id']);
        logger_write(
            eventType: 'reservation_unchanged',
            message: 'Reservation payload hash unchanged.',
            reservationId: (string) $existing['id'],
            externalId: $externalReservationId
        );

        return [
            'action' => 'unchanged',
            'reservation_id' => (int) $existing['id'],
            'external_reservation_id' => $externalReservationId,
        ];
    }

    $db->begin_transaction();

    try {
        $reservationId = save_reservation_record_set_to_local_database(db: $db, recordSet: $mapped);

        $action = detect_cancelled_reservation_from_payload(reservation: $reservationPayload)
            ? 'cancelled'
            : ($existing === null ? 'imported' : 'updated');
        $message = 'Reservation synchronized from context=' . $context . '.';

        write_reservation_audit_log(
            db: $db,
            reservationId: $reservationId,
            externalReservationId: $externalReservationId,
            eventType: $action,
            oldPayloadHash: $existing['payload_hash'] ?? null,
            newPayloadHash: $newPayloadHash,
            message: $message,
            payload: $reservationPayload
        );

        $db->commit();
    } catch (Throwable $exception) {
        $db->rollback();
        logger_write(eventType: 'reservation_error', message: $exception->getMessage(), reservationId: null, externalId: $externalReservationId);
        throw $exception;
    }

    logger_write(eventType: 'reservation_' . $action, message: $message, reservationId: (string) $reservationId, externalId: $externalReservationId);

    return [
        'action' => $action,
        'reservation_id' => $reservationId,
        'external_reservation_id' => $externalReservationId,
    ];
}

/**
 * Flattens one HotelSync reservation payload into local header, room rows, and rate plan rows.
 *
 * @param array<string, mixed> $reservation Raw reservation payload.
 * @param string $currency Fallback currency when the payload does not expose one.
 *
 * @return array{
 *     reservation:array{
 *         external_reservation_id:string,
 *         property_id:string,
 *         status:string,
 *         guest_status:string,
 *         guest_name:string,

 *         updated_at:string
 *     }>,

 *         external_rate_plan_id:string,
 *         rate_plan_name:string,
 *         meal_plan:string
 *     }>
 * }
 *
 * @throws RuntimeException When required upstream identifiers or dates are missing or malformed.
 */
function transform_reservation_payload_to_local_record_set(array $reservation, string $currency = 'EUR'): array
{
    $externalReservationId = app_require_non_empty_string(
        value: $reservation['id_reservations'] ?? '',
        label: 'HotelSync reservation id_reservations'
    );
    $propertyId = app_require_non_empty_string(
        value: $reservation['id_properties'] ?? '',
        label: 'HotelSync reservation id_properties'
    );
    $guestName = trim(string: (string) ($reservation['first_name'] ?? '') . ' ' . (string) ($reservation['last_name'] ?? ''));
    $guestName = $guestName !== '' ? $guestName : 'Unknown guest';
    $arrivalDate = app_require_iso_date_string(
        value: $reservation['date_arrival'] ?? '',
        label: 'HotelSync reservation date_arrival'
    );
    $departureDate = app_require_iso_date_string(
        value: $reservation['date_departure'] ?? '',
        label: 'HotelSync reservation date_departure'
    );
    $now = app_now();

    $mapped = [
        'reservation' => [
            'external_reservation_id' => $externalReservationId,
            'property_id' => $propertyId,
            'status' => (string) ($reservation['status'] ?? 'unknown'),
            'guest_status' => (string) ($reservation['guest_status'] ?? ''),
            'guest_name' => $guestName,
            'guest_email' => (string) ($reservation['email'] ?? ''),
            'guest_phone' => (string) ($reservation['phone'] ?? ''),
            'channel_name' => (string) ($reservation['channel_name'] ?? ''),
            'reference_code' => (string) ($reservation['reference'] ?? ''),
            'arrival_date' => $arrivalDate,
            'departure_date' => $departureDate,
            'nights' => (string) ((int) ($reservation['nights'] ?? 0)),
            'total_amount' => (string) ((float) ($reservation['total_price'] ?? 0)),
            'currency' => (string) ($reservation['currency'] ?? $currency),
            'lock_id' => 'LOCK-' . $externalReservationId . '-' . $arrivalDate,
            'payload_hash' => app_payload_hash(payload: $reservation),
            'raw_payload_json' => app_json_encode(value: $reservation),
            'cancelled_at' => (string) ($reservation['date_canceled'] ?? ''),
            'source_created_at' => (string) ($reservation['date_created'] ?? ''),
            'source_modified_at' => (string) ($reservation['date_modified'] ?? ''),
            'last_synced_at' => $now,
            'created_at' => $now,
            'updated_at' => $now,
        ],
        'rooms' => [],
        'rate_plans' => [],
    ];

    $ratePlans = [];
    $reservationRooms = is_array(value: ($reservation['rooms'] ?? null))
        ? $reservation['rooms']
        : [];

    // Rate plans are keyed by external id so one reservation snapshot cannot multiply the same plan locally.
    if (!empty($reservation['id_pricing_plans'])) {
        $ratePlans[(string) $reservation['id_pricing_plans']] = [
            'external_rate_plan_id' => (string) $reservation['id_pricing_plans'],
            'rate_plan_name' => (string) ($reservation['pricing_plan_name'] ?? 'Unknown rate plan'),
            'meal_plan' => '',
        ];
    }

    foreach ($reservationRooms as $room) {
        if (!is_array(value: $room)) {
            continue;
        }

        $mapped['rooms'][] = [
            'external_reservation_room_id' => (string) ($room['id_reservations_rooms'] ?? ''),
            'external_room_type_id' => (string) ($room['id_room_types'] ?? ''),
            'external_room_id' => (string) ($room['id_rooms'] ?? ''),
            'room_name' => (string) ($room['name'] ?? 'Unknown room'),
            'room_number' => (string) ($room['room_number'] ?? ''),
            'arrival_date' => (string) ($room['date_arrival'] ?? $mapped['reservation']['arrival_date']),
            'departure_date' => (string) ($room['date_departure'] ?? $mapped['reservation']['departure_date']),
            'nights_count' => (string) ((int) ($room['nights_count'] ?? 0)),
            'quantity' => '1',
            'adults' => (string) ((int) ($room['adults'] ?? 0)),
            'children_total' => (string) app_sum_children(payload: $room),
            'seniors' => (string) ((int) ($room['seniors'] ?? 0)),
            'total_guests' => (string) ((int) ($room['total_guests'] ?? 0)),
            'unit_amount' => (string) ((float) ($room['avg_price'] ?? 0)),
            'total_amount' => (string) ((float) ($room['total_price'] ?? 0)),
            'status' => (string) ($room['status'] ?? $mapped['reservation']['status']),
            'meal_plan' => (string) ($room['first_meal'] ?? ''),
            'raw_payload_json' => app_json_encode(value: $room),
            'created_at' => $now,
            'updated_at' => $now,
        ];

        $roomNights = is_array(value: ($room['nights'] ?? null))
            ? $room['nights']
            : [];

        foreach ($roomNights as $night) {
            if (!is_array(value: $night) || empty($night['id_pricing_plans'])) {
                continue;
            }

            $externalRatePlanId = (string) $night['id_pricing_plans'];
            $ratePlans[$externalRatePlanId] = [
                'external_rate_plan_id' => $externalRatePlanId,
                'rate_plan_name' => (string) ($night['pricing_plan_name'] ?? $reservation['pricing_plan_name'] ?? 'Unknown rate plan'),
                'meal_plan' => (string) ($room['first_meal'] ?? ''),
            ];
        }
    }

    $mapped['rate_plans'] = array_values(array: $ratePlans);

    return $mapped;
}

function process_reservation_webhook_event(string $rawPayload): array
{
    if (trim(string: $rawPayload) === '') {
        return [
            'status_code' => 400,
            'body' => [
                'status' => 'error',
                'message' => 'Webhook body is empty.',
            ],
        ];
    }

    try {
        $payload = app_json_decode(value: $rawPayload);
    } catch (RuntimeException $exception) {
        return [
            'status_code' => 400,
            'body' => [
                'status' => 'error',
                'message' => $exception->getMessage(),
            ],
        ];
    }

    if (!is_array(value: $payload)) {
        return [
            'status_code' => 400,
            'body' => [
                'status' => 'error',
                'message' => 'Webhook payload must be a JSON object.',
            ],
        ];
    }

    $context = transform_webhook_payload_to_webhook_context(payload: $payload);
    $externalReservationId = $context['external_reservation_id'];
    $embeddedReservation = $context['embedded_reservation'];

    if ($embeddedReservation === null && $externalReservationId === '') {
        return [
            'status_code' => 400,
            'body' => [
                'status' => 'error',
                'message' => 'Webhook payload does not include a reservation id.',
            ],
        ];
    }

    $db = db_connection();
    $existing = find_webhook_event_by_dedupe_key(db: $db, dedupeKey: $context['dedupe_key']);

    // A processed dedupe key is a hard stop: webhook replay must never reopen a completed state transition.
    if ($existing !== null && (int) $existing['is_processed'] === 1) {
        logger_write(eventType: 'webhook_duplicate', message: 'Duplicate webhook ignored.', reservationId: null, externalId: $externalReservationId);

        return [
            'status_code' => 200,
            'body' => [
                'status' => 'duplicate',
                'message' => 'Webhook already processed.',
            ],
        ];
    }

    if (
        $existing !== null
        && (int) $existing['is_processed'] === 0
        && ((string) ($existing['processing_error'] ?? '') === '')
    ) {
        logger_write(
            eventType: 'webhook_duplicate',
            message: 'Webhook already being processed.',
            reservationId: null,
            externalId: $externalReservationId
        );

        return [
            'status_code' => 200,
            'body' => [
                'status' => 'duplicate',
                'message' => 'Webhook is already being processed.',
            ],
        ];
    }

    // The inbox write happens before reservation sync so duplicate detection survives crashes and retries.
    $savedWebhook = $existing !== null
        ? [
            'id' => (int) $existing['id'],
            'was_inserted' => false,
        ]
        : save_webhook_event_in_inbox(
            db: $db,
            event: [
                'event_type' => $context['event_name'],
                'external_event_id' => $context['event_id'],
                'dedupe_key' => $context['dedupe_key'],
                'external_reservation_id' => $externalReservationId,
                'payload_hash' => $context['payload_hash'],
                'payload_json' => app_json_encode(value: $payload),
            ]
        );
    $webhookId = (int) $savedWebhook['id'];

    if ($existing === null && $savedWebhook['was_inserted'] === false) {
        logger_write(eventType: 'webhook_duplicate', message: 'Duplicate webhook ignored during inbox insert.', reservationId: null, externalId: $externalReservationId);

        return [
            'status_code' => 200,
            'body' => [
                'status' => 'duplicate',
                'message' => 'Webhook already processed or currently being handled.',
            ],
        ];
    }

    try {
        $result = $embeddedReservation !== null
            ? sync_reservation_payload_to_local_state(
                reservationPayload: $embeddedReservation,
                context: $context['sync_context'],
                currency: (string) ($embeddedReservation['currency'] ?? 'EUR')
            )
            : sync_reservation_from_hotelsync_by_id(
                externalReservationId: $externalReservationId,
                context: $context['sync_context']
            );

        // Only after the reservation write succeeds do we mark the inbox row as processed.
        mark_webhook_event_as_processed(db: $db, webhookId: $webhookId);

        logger_write(eventType: 'webhook_processed', message: 'Webhook processed successfully.', reservationId: null, externalId: $externalReservationId);

        return [
            'status_code' => 200,
            'body' => [
                'status' => 'ok',
                'action' => $result['action'],
                'reservation_id' => $result['external_reservation_id'],
            ],
        ];
    } catch (Throwable $exception) {
        mark_webhook_event_as_failed(db: $db, webhookId: $webhookId, errorMessage: $exception->getMessage());
        logger_write(eventType: 'webhook_failed', message: $exception->getMessage(), reservationId: null, externalId: $externalReservationId);

        return [
            'status_code' => 500,
            'body' => [
                'status' => 'error',
                'message' => 'Webhook processing failed.',
            ],
        ];
    }
}

<?php

declare(strict_types=1);

/**
 * Validates the transport-level webhook contract before business logic sees the payload.
 *
 * @param string $method HTTP method.
 * @param string|null $contentType Raw Content-Type header value.
 * @param int $payloadBytes Raw payload size in bytes.
 * @param int $maxPayloadBytes Configured maximum payload size.
 *
 * @return array{status_code:int,body:array<string, string>,headers:array<string, string>}|null
 */
function validate_webhook_request_transport(
    string $method,
    string|null $contentType,
    int $payloadBytes,
    int $maxPayloadBytes
): array|null {
    if (strtoupper(string: $method) !== 'POST') {
        return [
            'status_code' => 405,
            'body' => [
                'status' => 'error',
                'message' => 'Only POST is allowed.',
            ],
            'headers' => [
                'Allow' => 'POST',
            ],
        ];
    }

    if (!app_content_type_is_json(contentType: $contentType)) {
        return [
            'status_code' => 415,
            'body' => [
                'status' => 'error',
                'message' => 'Content-Type must be application/json.',
            ],
            'headers' => [],
        ];
    }

    if ($payloadBytes > $maxPayloadBytes) {
        return [
            'status_code' => 413,
            'body' => [
                'status' => 'error',
                'message' => 'Webhook payload is too large.',
            ],
            'headers' => [],
        ];
    }

    return null;
}

/**
 * Sends a JSON webhook response with the small set of headers this endpoint guarantees.
 *
 * @param int $statusCode HTTP status code.
 * @param array<string, string|int|float|bool|null> $body JSON response payload.
 * @param array<string, string> $headers Extra response headers.
 *
 * @return void
 */
function send_webhook_json_response(int $statusCode, array $body, array $headers = []): void
{
    http_response_code(response_code: $statusCode);

    if (app_get_correlation_id() !== '' && !array_key_exists(key: 'X-Correlation-Id', array: $headers)) {
        $headers['X-Correlation-Id'] = app_get_correlation_id();
    }

    foreach ($headers as $name => $value) {
        header(header: $name . ': ' . $value);
    }

    header(header: 'Content-Type: application/json');
    header(header: 'X-Content-Type-Options: nosniff');
    echo app_json_encode(value: $body);
}

/**
 * Handles the HTTP side of the webhook endpoint and keeps transport checks separate from business logic.
 *
 * @return void
 */
function receive_reservation_webhook_request(): void
{
    app_start_correlation_context(preferredCorrelationId: $_SERVER['HTTP_X_CORRELATION_ID'] ?? null);
    $maxPayloadBytes = (int) config_get(key: 'app.webhook.max_payload_bytes', default: 1048576);
    $transportError = validate_webhook_request_transport(
        method: (string) ($_SERVER['REQUEST_METHOD'] ?? 'GET'),
        contentType: $_SERVER['CONTENT_TYPE'] ?? '',
        payloadBytes: 0,
        maxPayloadBytes: $maxPayloadBytes
    );

    if ($transportError !== null) {
        send_webhook_json_response(
            statusCode: $transportError['status_code'],
            body: $transportError['body'],
            headers: $transportError['headers']
        );

        return;
    }

    $rawPayload = (string) file_get_contents(filename: 'php://input');
    $transportError = validate_webhook_request_transport(
        method: (string) ($_SERVER['REQUEST_METHOD'] ?? 'GET'),
        contentType: $_SERVER['CONTENT_TYPE'] ?? '',
        payloadBytes: strlen(string: $rawPayload),
        maxPayloadBytes: $maxPayloadBytes
    );

    if ($transportError !== null) {
        send_webhook_json_response(
            statusCode: $transportError['status_code'],
            body: $transportError['body'],
            headers: $transportError['headers']
        );

        return;
    }

    $result = process_reservation_webhook_event(rawPayload: $rawPayload);
    send_webhook_json_response(statusCode: (int) $result['status_code'], body: $result['body']);
}

<?php

declare(strict_types=1);

/**
 * Normalizes the incoming event name and falls back to a safe update-style label when the sender is vague.
 *
 * @param array<string, mixed> $payload Raw webhook payload.
 *
 * @return string
 */
function extract_webhook_event_name(array $payload): string
{
    return strtolower(string: app_optional_scalar_to_string(
        value: app_first_present(payload: $payload, paths: ['event_type', 'type', 'event', 'action'], default: 'reservation.updated'),
        default: 'reservation.updated'
    ));
}

/**
 * Extracts the provider event id when the payload offers one.
 *
 * @param array<string, mixed> $payload Raw webhook payload.
 *
 * @return string
 */
function extract_webhook_event_id(array $payload): string
{
    return app_optional_scalar_to_string(
        value: app_first_present(payload: $payload, paths: ['event_id', 'external_event_id', 'webhook_event_id'], default: ''),
        default: ''
    );
}

/**
 * Extracts the HotelSync reservation id from the common nesting patterns seen in webhook payloads.
 *
 * @param array<string, mixed> $payload Raw webhook payload.
 *
 * @return string
 */
function extract_reservation_id_from_webhook(array $payload): string
{
    return app_optional_scalar_to_string(value: app_first_present(
        payload: $payload,
        paths: [
            'reservation_id',
            'id_reservations',
            'reservation.id_reservations',
            'data.id_reservations',
            'data.reservation_id',
            'data.reservation.id_reservations',
        ],
        default: ''
    ), default: '');
}

/**
 * Returns the embedded reservation payload when the webhook already carries the full reservation snapshot.
 *
 * @param array<string, mixed> $payload Raw webhook payload.
 *
 * @return array<string, mixed>|null
 */
function extract_reservation_payload_from_webhook(array $payload): array|null
{
    if (isset($payload['reservation']) && is_array(value: $payload['reservation'])) {
        return $payload['reservation'];
    }

    if (isset($payload['data']['reservation']) && is_array(value: $payload['data']['reservation'])) {
        return $payload['data']['reservation'];
    }

    if (isset($payload['data']) && is_array(value: $payload['data']) && isset($payload['data']['id_reservations'], $payload['data']['rooms'])) {
        return $payload['data'];
    }

    if (isset($payload['id_reservations'], $payload['rooms'])) {
        return $payload;
    }

    return null;
}

/**
 * Builds the dedupe key from an explicit event id when possible, or from the payload hash when it is not.
 *
 * @param array<string, mixed> $payload Raw webhook payload.
 * @param string $payloadHash Canonical payload hash.
 *
 * @return string
 */
function build_webhook_dedupe_key(array $payload, string $payloadHash): string
{
    $eventId = extract_webhook_event_id(payload: $payload);

    return $eventId !== '' ? 'event:' . $eventId : 'hash:' . $payloadHash;
}

/**
 * Translates an event label into the local sync context string used by audit logs and processors.
 *
 * @param string $eventName Normalized webhook event name.
 *
 * @return string
 */
function map_webhook_event_name_to_reservation_sync_context(string $eventName): string
{
    return match (true) {
        str_contains(haystack: $eventName, needle: 'cancel') => 'webhook_cancel',
        str_contains(haystack: $eventName, needle: 'create'), str_contains(haystack: $eventName, needle: 'new') => 'webhook_create',
        default => 'webhook_update',
    };
}

/**
 * Transforms one raw webhook payload into the normalized context needed by dedupe and reservation processing.
 *
 * @param array<string, mixed> $payload Raw webhook payload.
 *
 * @return array{
 *     event_name:string,
 *     event_id:string,
 *     payload_hash:string,
 *     dedupe_key:string,
 *     external_reservation_id:string,
 *     embedded_reservation:array<string, mixed>|null,
 *     sync_context:string
 * }
 */
function transform_webhook_payload_to_webhook_context(array $payload): array
{
    $eventName = extract_webhook_event_name(payload: $payload);
    $payloadHash = app_payload_hash(payload: $payload);

    return [
        'event_name' => $eventName,
        'event_id' => extract_webhook_event_id(payload: $payload),
        'payload_hash' => $payloadHash,
        'dedupe_key' => build_webhook_dedupe_key(payload: $payload, payloadHash: $payloadHash),
        'external_reservation_id' => extract_reservation_id_from_webhook(payload: $payload),
        'embedded_reservation' => extract_reservation_payload_from_webhook(payload: $payload),
        'sync_context' => map_webhook_event_name_to_reservation_sync_context(eventName: $eventName),
    ];
}

CREATE TABLE IF NOT EXISTS rooms (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    external_room_type_id VARCHAR(64) NOT NULL,
    property_id VARCHAR(64) NOT NULL,
    name VARCHAR(255) NOT NULL,
    short_name VARCHAR(100) DEFAULT NULL,
    room_kind VARCHAR(100) DEFAULT NULL,
    occupancy INT UNSIGNED NOT NULL DEFAULT 0,
    base_price DECIMAL(12, 2) DEFAULT NULL,
    room_count INT UNSIGNED NOT NULL DEFAULT 0,
    local_code VARCHAR(255) NOT NULL,
    payload_hash CHAR(64) NOT NULL,
    raw_payload_json JSON NOT NULL,
    synced_at DATETIME NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    UNIQUE KEY uq_rooms_external_room_type_id (external_room_type_id),
    UNIQUE KEY uq_rooms_local_code (local_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS rate_plans (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    external_rate_plan_id VARCHAR(64) NOT NULL,
    property_id VARCHAR(64) NOT NULL,
    board_id VARCHAR(64) DEFAULT NULL,
    name VARCHAR(255) NOT NULL,
    meal_plan VARCHAR(120) NOT NULL,
    pricing_type VARCHAR(50) DEFAULT NULL,
    local_code VARCHAR(255) NOT NULL,
    payload_hash CHAR(64) NOT NULL,
    raw_payload_json JSON NOT NULL,
    synced_at DATETIME NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    UNIQUE KEY uq_rate_plans_external_rate_plan_id (external_rate_plan_id),
    UNIQUE KEY uq_rate_plans_local_code (local_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS reservations (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    external_reservation_id VARCHAR(64) NOT NULL,
    property_id VARCHAR(64) NOT NULL,
    status VARCHAR(50) NOT NULL,
    guest_status VARCHAR(50) DEFAULT NULL,
    guest_name VARCHAR(255) NOT NULL,
    guest_email VARCHAR(255) DEFAULT NULL,
    guest_phone VARCHAR(100) DEFAULT NULL,
    channel_name VARCHAR(120) DEFAULT NULL,
    reference_code VARCHAR(120) DEFAULT NULL,
    arrival_date DATE NOT NULL,
    departure_date DATE NOT NULL,
    nights INT UNSIGNED NOT NULL DEFAULT 0,
    total_amount DECIMAL(12, 2) NOT NULL DEFAULT 0,
    currency CHAR(3) NOT NULL DEFAULT 'EUR',
    lock_id VARCHAR(255) NOT NULL,
    payload_hash CHAR(64) NOT NULL,
    raw_payload_json JSON NOT NULL,
    cancelled_at DATETIME DEFAULT NULL,
    source_created_at DATETIME DEFAULT NULL,
    source_modified_at DATETIME DEFAULT NULL,
    last_synced_at DATETIME NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    UNIQUE KEY uq_reservations_external_reservation_id (external_reservation_id),
    UNIQUE KEY uq_reservations_lock_id (lock_id),
    KEY idx_reservations_arrival_departure (arrival_date, departure_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS reservation_rooms (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    reservation_id INT UNSIGNED NOT NULL,
    room_id INT UNSIGNED DEFAULT NULL,
    external_reservation_room_id VARCHAR(64) NOT NULL,
    external_room_type_id VARCHAR(64) DEFAULT NULL,
    external_room_id VARCHAR(64) DEFAULT NULL,
    room_name VARCHAR(255) NOT NULL,
    room_number VARCHAR(100) DEFAULT NULL,
    arrival_date DATE NOT NULL,
    departure_date DATE NOT NULL,
    nights_count INT UNSIGNED NOT NULL DEFAULT 0,
    quantity INT UNSIGNED NOT NULL DEFAULT 1,
    adults INT UNSIGNED NOT NULL DEFAULT 0,
    children_total INT UNSIGNED NOT NULL DEFAULT 0,
    seniors INT UNSIGNED NOT NULL DEFAULT 0,
    total_guests INT UNSIGNED NOT NULL DEFAULT 0,
    unit_amount DECIMAL(12, 2) NOT NULL DEFAULT 0,
    total_amount DECIMAL(12, 2) NOT NULL DEFAULT 0,
    status VARCHAR(50) DEFAULT NULL,
    meal_plan VARCHAR(120) DEFAULT NULL,
    raw_payload_json JSON NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    UNIQUE KEY uq_reservation_rooms_external_reservation_room_id (external_reservation_room_id),
    KEY idx_reservation_rooms_reservation_id (reservation_id),
    CONSTRAINT fk_reservation_rooms_reservation_id FOREIGN KEY (reservation_id) REFERENCES reservations (id) ON DELETE CASCADE,

CREATE TABLE IF NOT EXISTS reservation_rate_plans (

CREATE TABLE IF NOT EXISTS audit_logs (

CREATE TABLE IF NOT EXISTS invoice_queue (

    created_at DATETIME NOT NULL,

    CONSTRAINT fk_invoice_queue_reservation_id FOREIGN KEY (reservation_id) REFERENCES reservations (id) ON DELETE CASCADE

<?php

declare(strict_types=1);

$GLOBALS['__tests'] = [
    'passed' => 0,
    'failed' => 0,
];

/**
 * Runs one test case and records the result in a tiny in-memory counter.
 *
 * @param string $name Human-readable test name.
 * @param callable $callback Test body.
 *
 * @return void
 */
function test_case(string $name, callable $callback): void
{
    try {
        $callback();
        $GLOBALS['__tests']['passed']++;
        echo '[PASS] ' . $name . PHP_EOL;
    } catch (Throwable $exception) {
        $GLOBALS['__tests']['failed']++;
        echo '[FAIL] ' . $name . ' - ' . $exception->getMessage() . PHP_EOL;
    }
}

/**
 * Asserts strict equality and prints both values when they drift apart.
 *
 * @param mixed $expected Expected value.
 * @param mixed $actual Actual value.
 * @param string $message Extra failure detail.
 *
 * @return void
 */
function assert_same(mixed $expected, mixed $actual, string $message = ''): void
{
    if ($expected !== $actual) {
        throw new RuntimeException(
            message: ($message !== '' ? $message . ' ' : '')
            . 'Expected ' . var_export(value: $expected, return: true)
            . ' but got ' . var_export(value: $actual, return: true) . '.'
        );
    }
}

/**
 * Asserts that a condition is true and throws a readable message when it is not.
 *
 * @param bool $condition Condition under test.
 * @param string $message Failure message.
 *
 * @return void
 */
function assert_true(bool $condition, string $message): void
{
    if ($condition !== true) {
        throw new RuntimeException(message: $message);
    }
}

/**
 * Asserts that a callable throws the expected exception and optionally checks the message fragment.
 *
 * @param callable $callback Code that should throw.
 * @param string $exceptionClass Expected exception class name.
 * @param string $messageContains Optional message fragment.
 *
 * @return Throwable
 */
function assert_throws(callable $callback, string $exceptionClass, string $messageContains = ''): Throwable
{
    try {
        $callback();
    } catch (Throwable $exception) {
        if (!$exception instanceof $exceptionClass) {
            throw new RuntimeException(
                message: 'Expected exception ' . $exceptionClass . ' but got ' . $exception::class . '.'
            );
        }

        if ($messageContains !== '' && !str_contains(haystack: $exception->getMessage(), needle: $messageContains)) {
            throw new RuntimeException(
                message: 'Expected exception message to contain ' . var_export(value: $messageContains, return: true)
                . ' but got ' . var_export(value: $exception->getMessage(), return: true) . '.'
            );
        }

        return $exception;
    }

    throw new RuntimeException(message: 'Expected exception ' . $exceptionClass . ' but nothing was thrown.');
}

/**
 * Prints the final test summary and exits with a failing code when any test failed.
 *
 * @return void
 */
function test_summary(): void
{
    $passed = (int) $GLOBALS['__tests']['passed'];
    $failed = (int) $GLOBALS['__tests']['failed'];

    echo PHP_EOL . 'Summary: passed=' . $passed . ' failed=' . $failed . PHP_EOL;

    if ($failed > 0) {
        exit(1);
    }
}

<?php

declare(strict_types=1);

/**
 * Runs one callback with temporary environment values and restores the previous process state afterwards.
 *
 * @param array<string, string|null> $overrides Environment values to set for the callback.
 * @param callable $callback Callback executed under the temporary environment.
 *
 * @return mixed
 */
function test_with_env(array $overrides, callable $callback): mixed
{
    $previousValues = [];

    foreach ($overrides as $key => $value) {
        $currentValue = getenv(name: $key);
        $previousValues[$key] = $currentValue === false ? null : $currentValue;

        if ($value === null) {
            putenv(assignment: $key);
            unset($_ENV[$key], $_SERVER[$key]);
        } else {
            putenv(assignment: $key . '=' . $value);
            $_ENV[$key] = $value;
            $_SERVER[$key] = $value;
        }
    }

    app_reset_config_cache();

    try {
        return $callback();
    } finally {
        foreach ($previousValues as $key => $value) {
            if ($value === null) {
                putenv(assignment: $key);
                unset($_ENV[$key], $_SERVER[$key]);
            } else {
                putenv(assignment: $key . '=' . $value);
                $_ENV[$key] = $value;
                $_SERVER[$key] = $value;
            }
        }

        http_override_post_json_transport(transport: null);
        app_reset_config_cache();
    }
}

/**
 * Requires every test file below one root directory in a stable order so nested folders stay easy to organize.
 *
 * @param string $rootDirectory Absolute test root path.
 *
 * @return void
 */
function require_test_files_from_directory(string $rootDirectory): void
{
    $testFiles = [];
    $iterator = new RecursiveIteratorIterator(
        new RecursiveDirectoryIterator($rootDirectory, FilesystemIterator::SKIP_DOTS)
    );

    foreach ($iterator as $file) {
        if (!$file->isFile()) {
            continue;
        }

        if (!str_ends_with($file->getFilename(), '_test.php')) {
            continue;
        }

        $testFiles[] = $file->getPathname();
    }

    sort($testFiles);

    foreach ($testFiles as $testFile) {
        require $testFile;
    }
}
