# Quality Rules

This file defines the questions and standards that every non-trivial item must satisfy.

## Required Questions

Every idea, backlog item, plan, architecture note, business note, and
release-impacting change must answer:

1. What outcome should the user or operator achieve?
2. What should the system do to make that outcome possible and safe?
3. What should be visible or externally observable if the change works?
4. How will correctness, risk, and rollback be verified?

## Rules

- Write the outcome first and the technical detail second.
- Keep ideas and plans in plain language before moving into implementation.
- If an item cannot answer the required questions, it is not ready.
- Specialized product questions belong in child-repo overlays, not in this file.
- When quality rules change, update the owning document in the same work item.
- These questions are mandatory in governance, business, planning, execution, and review docs.

## Current Policy

No active exceptions.
